# Galyarder Initial Linear Backlog

## Purpose
This is the first backlog proposal derived from the digital company vision.

These items should be turned into actual Linear projects and issues.

## Project 1 - Founder Office Buildout
Goal:
Make the founder office a real department with clean skill mapping, reporting, and execution standards.

Suggested issues:
- finalize founder office reporting templates
- define founder office recurring metrics
- map `fundraising-operator` outputs to Obsidian reports
- define founder office Linear workflow
- audit additional founder / capital skills from external repos

## Project 2 - Engineering Department Audit
Goal:
Benchmark current engineering department against stronger external skill libraries and identify upgrades.

Suggested issues:
- audit architecture skills vs external benchmarks
- audit advanced code review and adversarial review gaps
- audit Playwright / test automation gaps
- audit CI/CD and observability skill gaps
- propose import / adapt / replace plan for engineering department

## Project 3 - Growth Department Audit
Goal:
Turn growth into a stronger department rather than a scattered collection of tactics.

Suggested issues:
- map current growth agents to current skills
- identify orphan growth skills
- audit external growth / marketing skill libraries
- define growth department report template
- define growth Linear workflow

## Project 4 - Obsidian Reporting System
Goal:
Make Obsidian the official founder reporting and company memory layer.

Suggested issues:
- define founder master report template
- define department report templates
- define milestone summary template
- define decision-log template
- define report cadence rules by department

## Project 5 - Agent-to-Skill Mapping Rollout
Goal:
Make every major department lead explicitly backed by a mapped skill team.

Suggested issues:
- complete executive / founder-office mapping
- complete engineering mapping
- complete growth mapping
- complete security mapping
- complete legal-finance mapping
- complete knowledge / obsidian mapping
- identify orphan skills with no manager

## Project 6 - Documentation Rewrite
Goal:
Rewrite framework docs around the digital company model.

Suggested issues:
- rewrite README narrative toward digital company framing
- rewrite AGENTS docs to reflect department ownership
- define department manifests
- update host bootstrap docs where useful

## Project 7 - External Skill Hiring Pipeline
Goal:
Create a repeatable process for auditing and importing high-value external skills.

Suggested issues:
- define import criteria
- define adapt vs import vs reject rubric
- create benchmark list of external repositories
- start with `claude-skills` engineering domain audit
- continue with `claude-skills` growth domain audit

## Priority Order
Recommended order:
1. Obsidian Reporting System
2. Agent-to-Skill Mapping Rollout
3. Engineering Department Audit
4. Growth Department Audit
5. Founder Office Buildout refinement
6. Documentation Rewrite
7. External Skill Hiring Pipeline

## Founder-Facing Outcome
When this backlog is executed well, the founder should experience:
- clearer delegation
- cleaner reports
- less context-switching
- stronger departments
- better imported skills where they matter
